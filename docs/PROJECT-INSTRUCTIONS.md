КОНТЕКСТ
Ова е проект за целосен редизајн на јавната веб-страница financebuddy.mk — сметководствена/
консалтинг агенција во Скопје. Трговско име: FinanceBuddy.mk (се користи насекаде). Правно име:
Друштво за трговија и услуги ФАЈНЕНС БАДИ ДООЕЛ Скопје (се користи САМО во footer copyright нота
и во LocalBusiness schema markup — никаде во UI, навигација или маркетинг текст).

Постои и одделен производ — клиентски портал на portal.financebuddy.mk (Laravel + Inertia + Vue 3),
веќе во развој. Овој проект НЕ е порталот — тоа е целосно посебна апликација, посебна база,
посебен codebase, истиот сервер (DigitalOcean дроплет). Никогаш не мешај ги двата проекта.

ИЗВОР НА ВИСТИНА
Целосна спецификација постои во папката /docs (фајлови 00 до 10 — OVERVIEW, CONTENT-AUDIT,
TECH-STACK, CODING-STANDARDS, LOGGING-MONITORING, DESIGN-SYSTEM, SEO-STRATEGY, SITE-STRUCTURE,
CMS-ADMIN, DEPLOYMENT, ROADMAP). На почеток на секоја сесија, прочитај го релевантниот документ
пред да продолжиш со работа — не претпоставувај, провери во документацијата. Ако нешто во
разговорот противречи на документацијата, прашај наместо тивко да отстапиш, и ажурирај го
документот штом се донесе нова одлука (документацијата треба да остане жива, не статична).

ТЕХНИЧКИ СТЕК (детали во 02-TECH-STACK.md)
Laravel 11 + Inertia.js + Vue 3 (Composition API, <script setup>) + Tailwind CSS + Vite.
MySQL база. Истиот стек како порталот, заради конзистентност на знаење и конвенции.

ПРАВИЛА НА КОДИРАЊЕ (целосно во 03-CODING-STANDARDS.md — ова е приоритетен документ, важи за
секоја линија код што се пишува)
- Тенки контролери, Form Request валидација, Eloquent relationships експлицитни
- Vue: секогаш <script setup>, route() helper наместо hardcoded URL-ови, компоненти под ~200 линии
- Целосни преработки на мали/средни фајлови наместо парцијални закрпи (воспоставена навика)
- Сите кориснички видливи стрингови на македонски
- Tailwind утилити класи, дизајн токени централно во tailwind.config.js — никогаш hardcoded хекс бои
- Без raw SQL со interpolated input; contact пораки никогаш не се рендерираат како HTML
- CSRF заштита, rate limiting на форми секогаш активни
- Commit пораки на англиски, мали и чести commits, .env никогаш во git

ДИЗАЈН (целосно во 05-DESIGN-SYSTEM.md)
Топла, човечка естетика — НЕ генеричка "fintech startup" естетика (без студена темно-сина,
без glassmorphism). Палета: портокалова #FF6600 (примарна, наследена од порталот), топла
крем-бела позадина (не чисто бело), темно зелена како секундарна боја. Типографија: Fraunces
(naslovi), Inter (текст), JetBrains Mono (бројки/статистики). Сигнатура елемент: рачно-нацртан
SVG подвлекувач во клучни зборови од насловите. Секогаш провери: дали ова изгледа како секој
друг финансиски сајт, или носи карактер специфичен за FinanceBuddy?

SEO (целосно во 06-SEO-STRATEGY.md — некомпромисно, секоја страница мора да го исполнува ова)
Уникатен title/meta description по страница, og:locale секогаш mk_MK (не en_US — грешка во старата
верзија), html lang="mk", JSON-LD Organization/LocalBusiness schema, Article schema на блог
постови, FAQPage schema на FAQ. Секоја услуга добива своја URL (/uslugi/{slug}), не сите водат кон
контакт формата како во стариот сајт. Постоечките blog URL-ови (FB#1-20+, целосна листа во
01-CONTENT-AUDIT.md) МОРААТ да добијат точни 301 redirects при миграција — никогаш не се губи
индексирана стара URL без redirect.
