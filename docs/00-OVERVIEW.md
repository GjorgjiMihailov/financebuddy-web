## 1. Контекст

FinanceBuddy.mk е сметководствена и консалтинг агенција со седиште во Скопје. Правното име на
друштвото е **Друштво за трговија и услуги ФАЈНЕНС БАДИ ДООЕЛ Скопје**, но во целата комуникација
(маркетинг, социјални мрежи, документи свртени кон клиенти) се користи исклучиво трговското име
**FinanceBuddy.mk**. Правното име се користи само таму каде законски е задолжително (фактури,
договори, импресум/footer ситна нота, регистарски податоци).

Компанијата има два дигитални производи кои треба да остане јасно одвоени:

| Производ | Домен | Намена | Стек |
|---|---|---|---|
| **Клиентски портал** | `portal.financebuddy.mk` | Логиран пристап за клиенти: фактури, сметководство, документи | Laravel + Inertia.js + Vue 3 + MySQL/PostgreSQL, веќе во развој |
| **Јавна веб-страница** | `financebuddy.mk` | Маркетинг, презентација, блог, лидови (предмет на овој документ) | Нов: Laravel + Inertia.js + Vue 3 (ист стек, различна апликација) |

Двата производи живеат на **истиот DigitalOcean дроплет**, како посебни Laravel апликации (секоја
со свој Nginx server block / vhost, своја база, свој deploy циклус), не како еден заеднички
codebase. Ова е намерна одлука — јавната страница мора да остане лесна и брза, а порталот има
сосема различни безбедносни и функционални барања (автентикација на клиенти, финансиски податоци).

## 2. Сегашна состојба (нешто за бришење)

Тековниот сајт е **WordPress + Elementor** (page builder), со WPForms за контакт формата.
Содржински сајтот ги има следните делови, кои ги пописавме во `01-CONTENT-AUDIT.md`:

- Дома (хиро секција, услуги preview, "за нас" preview, статистики, testimonials, контакт форма,
  blog preview, newsletter формa)
- За нас
- Услуги (8 услуги наброени)
- FinanceBlog (категории: HR and Leadership, и веројатно други — потребна целосна инвентаризација)
- Контакт (форма + контакт детали)
- FAQ (прашања "од пракса")
- Кариера

**Проблеми со тековното решение** (причини зошто се менува целосно):

1. **Перформанси** — Elementor генерира тежок, нечист HTML/CSS/JS; бавно вчитување штети на Core
   Web Vitals, кои директно влијаат на Google рангирање.
2. **SEO таван** — page builder-ите тешко дозволуваат фина контрола над structured data
   (JSON-LD schema), canonical логика, internal linking стратегија, и сл.
3. **Безбедност и одржување** — WordPress плагин екосистем е чест извор на пропусти; бара
   постојани updates.
4. **Недоследен дизајн** — генерички Elementor темплејти, нема силен визуелен идентитет.
5. **Нема контрола врз backend** — Гјоргји сака сопствен admin панел за блог и контакт пораки,
   без зависност од WP admin / плагини.

## 3. Цели на новиот проект

1. **Модерен, атрактивен дизајн** — силен визуелен идентитет, не генерички темплејт (видете
   `05-DESIGN-SYSTEM.md`).
2. **Целосна SEO оптимизација** — техничко + содржинско SEO од темел (видете
   `06-SEO-STRATEGY.md`).
3. **Брзина** — таргет Core Web Vitals "Good" во сите категории (LCP < 2.5s, INP < 200ms,
   CLS < 0.1).
4. **Сопствен backend / admin панел** за:
   - Пишување, уредување и објавување блог постови (со SEO полиња по пост)
   - Преглед на пораки од контакт формата (со статус: ново / прочитано / одговорено)
5. **Чиста, одржлива кодна база** која Гјоргји може сам да ја проширува со помош на Claude Code,
   следејќи конзистентни конвенции (видете `03-CODING-STANDARDS.md`).
6. **Правилно логирање и мониторинг** на грешки, контакт submissions, и админ активности
   (видете `04-LOGGING-MONITORING.md`).
7. **Зачувување на содржинската вредност** — постоечките блог постови (URL-ови, наслови) не смее
   да се изгубат; потребен е 301 redirect план при миграција (видете `06-SEO-STRATEGY.md`,
   секција "Миграција и редиректи").

## 4. Што НЕ е дел од овој проект (за сега)

- Клиентскиот портал (`portal.financebuddy.mk`) — тој е посебен проект, веќе во тек.
- E-commerce / плаќање онлајн.
- Мултијазичност (англиски верзија) — не е барано во почетна фаза, но архитектурата ќе остане
  отворена за тоа (видете напомена во `02-TECH-STACK.md`).
- Контролирана е само јавната содржина; внатрешните алатки на УЈП и другите проекти на Гјоргји
  (SCOUTED.MK, TJ ProSports, footballstats-mk) не се поврзани со овој проект.

## 7. Како да се користи оваа документација во Claude Code / Cowork

Препорачан редослед:

1. Стави ја целата `financebuddy-docs/` папка во root на новиот Laravel проект (на пр. како
   `/docs` папка) — Claude Code ќе може да ја чита при секоја сесија.
2. Започни со `02-TECH-STACK.md` и `09-DEPLOYMENT.md` за иницијален `laravel new` setup.
3. Потоа `03-CODING-STANDARDS.md` — стави го како `CLAUDE.md` во root на проектот (Claude Code
   автоматски го чита тој фајл при секоја сесија — видете напомена во тој документ).
4. Гради по редослед: `07-SITE-STRUCTURE.md` → статични страници → `08-CMS-ADMIN.md` → блог.
5. На крај: `06-SEO-STRATEGY.md` checklist пред лансирање.

Секоја сесија со Claude Code треба да почне со кратка реченица од типот: *"Прочитај
/docs/03-CODING-STANDARDS.md и /docs/02-TECH-STACK.md пред да продолжиш"* — ова го одржува
кодот конзистентен низ сесии.
